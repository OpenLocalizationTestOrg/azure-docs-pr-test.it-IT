## <a name="load-balancing-methods"></a><span data-ttu-id="b9621-109">Metodi di bilanciamento del carico.</span><span class="sxs-lookup"><span data-stu-id="b9621-109">Load Balancing Methods</span></span>
<span data-ttu-id="b9621-110">Gestione traffico di Azure utilizza tre metodi diversi per il bilanciamento del carico.</span><span class="sxs-lookup"><span data-stu-id="b9621-110">Azure Traffic Manager uses three different load balancing methods.</span></span> <span data-ttu-id="b9621-111">Questi metodi vengono descritti nell'elenco seguente, in quanto pertinenti alle app Web di Azure.</span><span class="sxs-lookup"><span data-stu-id="b9621-111">These are described  in the following list as they pertain to Azure web apps.</span></span>

* <span data-ttu-id="b9621-112">**Failover**: se sono presenti cloni di app Web in aree diverse, è possibile utilizzare questo metodo per configurare un'app Web in modo da gestire tutto il traffico dei client Web e un'altra, in un'area diversa, per gestire tale traffico nel caso in cui il primo sito Web risulti non disponibile.</span><span class="sxs-lookup"><span data-stu-id="b9621-112">**Failover**: If you have web app clones in different regions, you can use this method to configure one web app to service all web client traffic, and configure another web app in a different region to service that traffic in case the first web app becomes unavailable.</span></span>
* <span data-ttu-id="b9621-113">**Round Robin**: se sono presenti cloni di app Web in aree diverse, è possibile utilizzare questo metodo per distribuire equamente il traffico tra app Web in aree diverse.</span><span class="sxs-lookup"><span data-stu-id="b9621-113">**Round Robin**: If you have web app clones in different regions, you can use this method to distribute traffic equally across the web apps in different regions.</span></span>
