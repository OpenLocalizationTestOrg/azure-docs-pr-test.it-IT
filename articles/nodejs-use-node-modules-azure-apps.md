## <a name="nodejs-modules"></a><span data-ttu-id="20a3c-112">Moduli Node.js</span><span class="sxs-lookup"><span data-stu-id="20a3c-112">Node.js Modules</span></span>
<span data-ttu-id="20a3c-113">I moduli sono pacchetti JavaScript caricabili che forniscono funzionalità specifiche per l'applicazione.</span><span class="sxs-lookup"><span data-stu-id="20a3c-113">Modules are loadable JavaScript packages that provide specific functionality for your application.</span></span> <span data-ttu-id="20a3c-114">I moduli vengono in genere installati usando lo strumento da riga di comando **npm**, tuttavia, alcuni moduli, ad esempio il modulo HTTP, sono forniti nel pacchetto Node.js di base.</span><span class="sxs-lookup"><span data-stu-id="20a3c-114">Modules are usually installed using the **npm** command-line tool, however some modules (such as the http module) are provided as part of the core Node.js package.</span></span>

<span data-ttu-id="20a3c-115">Quando i moduli sono installati, vengono archiviati nella directory **node\_modules** nella radice della struttura di directory dell'applicazione.</span><span class="sxs-lookup"><span data-stu-id="20a3c-115">When modules are installed, they are stored in the **node\_modules** directory at the root of your application directory structure.</span></span> <span data-ttu-id="20a3c-116">Ogni modulo nella directory **node\_modules** mantiene la propria directory **node\_modules** che contiene i moduli da cui dipende e questo comportamento si ripete per tutti i moduli fino alla fine della catena delle dipendenze.</span><span class="sxs-lookup"><span data-stu-id="20a3c-116">Each module within the **node\_modules** directory maintains its own **node\_modules** directory that contains any modules that it depends on, and this behavior repeats for every module all the way down the dependency chain.</span></span> <span data-ttu-id="20a3c-117">Questo ambiente consente a ogni modulo installato di avere requisiti di versione specifici per i moduli da cui dipende, anche se ciò può determinare una struttura di directory molto grande.</span><span class="sxs-lookup"><span data-stu-id="20a3c-117">This environment allows each module installed to have its own version requirements for the modules it depends on, however it can result in quite a large directory structure.</span></span>

<span data-ttu-id="20a3c-118">La distribuzione della directory **node\_modules** come parte dell'applicazione aumenta le dimensioni della distribuzione rispetto a quando si usa un file **package.json** o **npm-shrinkwrap.json**, ma questa soluzione garantisce che le versioni dei moduli usate in fase di produzione siano le stesse dei moduli usati in fase di sviluppo.</span><span class="sxs-lookup"><span data-stu-id="20a3c-118">Deploying the **node\_modules** directory as part of your application increases the size of the deployment when compared to using a **package.json** or **npm-shrinkwrap.json** file; however, it does guarantee that the versions of the modules used in production are the same as the modules used in development.</span></span>

### <a name="native-modules"></a><span data-ttu-id="20a3c-119">Moduli nativi</span><span class="sxs-lookup"><span data-stu-id="20a3c-119">Native Modules</span></span>
<span data-ttu-id="20a3c-120">Sebbene la maggior parte dei moduli sia semplicemente costituita da file JavaScript in testo normale, alcuni di essi sono immagini binarie specifiche della piattaforma.</span><span class="sxs-lookup"><span data-stu-id="20a3c-120">While most modules are simply plain-text JavaScript files, some modules are platform-specific binary images.</span></span> <span data-ttu-id="20a3c-121">Questi moduli vengono compilati in fase di installazione, in genere tramite Python e node-gyp.</span><span class="sxs-lookup"><span data-stu-id="20a3c-121">These modules are compiled at install time, usually by using Python and node-gyp.</span></span> <span data-ttu-id="20a3c-122">Poiché Servizi cloud di Azure si basa sulla distribuzione della cartella **node\_modules** come parte dell'applicazione, qualsiasi modulo nativo incluso come parte dei moduli installati può funzionare in un servizio cloud purché sia stato installato e compilato in un sistema di sviluppo Windows.</span><span class="sxs-lookup"><span data-stu-id="20a3c-122">Since Azure Cloud Services rely on the **node\_modules** folder being deployed as part of the application, any native module included as part of the installed modules should work in a cloud service as long as it was installed and compiled on a Windows development system.</span></span>

<span data-ttu-id="20a3c-123">Servizio app di Azure non supporta tutti i moduli nativi e potrebbe non riuscire a compilare i moduli con prerequisiti specifici.</span><span class="sxs-lookup"><span data-stu-id="20a3c-123">Azure App Service does not support all native modules, and might fail when compiling modules with specific prerequisites.</span></span> <span data-ttu-id="20a3c-124">Mentre alcuni moduli più diffusi, come MongoDB, hanno dipendenze native facoltative e funzionano anche senza di esse, due soluzioni alternative hanno fornito risultati positivi con quasi tutti i moduli nativi disponibili attualmente:</span><span class="sxs-lookup"><span data-stu-id="20a3c-124">While some popular modules like MongoDB have optional native dependencies and work fine without them, two workarounds proved successful with almost all native modules available today:</span></span>

* <span data-ttu-id="20a3c-125">Eseguire **npm install** in un computer Windows con installati tutti i prerequisiti del modulo nativo.</span><span class="sxs-lookup"><span data-stu-id="20a3c-125">Run **npm install** on a Windows machine that has all the native module's prerequisites installed.</span></span> <span data-ttu-id="20a3c-126">Distribuire quindi la cartella **node\_modules** creata come parte dell'applicazione in Servizio app di Azure.</span><span class="sxs-lookup"><span data-stu-id="20a3c-126">Then, deploy the created **node\_modules** folder as part of the application to Azure App Service.</span></span>

  * <span data-ttu-id="20a3c-127">Prima di eseguire la compilazione, verificare che l'installazione locale di Node.js disponga di un'architettura corrispondente e che la versione sia molto simile a quella usata in Azure (i valori correnti possono essere verificati in fase di esecuzione tramite le proprietà **process.arch** e **process.version**).</span><span class="sxs-lookup"><span data-stu-id="20a3c-127">Before compiling, check that your local Node.js installation has matching architecture and the version is as close as possible to the one used in Azure (the current values can be checked on runtime from properties **process.arch** and **process.version**).</span></span>

* <span data-ttu-id="20a3c-128">È possibile configurare Servizio app di Azure per l'esecuzione di script bash o della shell durante la distribuzione, offrendo la possibilità di eseguire comandi personalizzati e di configurare con precisione la modalità di esecuzione di **npm install** .</span><span class="sxs-lookup"><span data-stu-id="20a3c-128">Azure App Service can be configured to execute custom bash or shell scripts during deployment, giving you the opportunity to execute custom commands and precisely configure the way **npm install** is being run.</span></span> <span data-ttu-id="20a3c-129">Per un video che illustra come configurare tale ambiente, vedere [Custom Website Deployment Scripts with Kudu] (Script di distribuzione di un sito Web personalizzato con Kudu).</span><span class="sxs-lookup"><span data-stu-id="20a3c-129">For a video showing how to configure that environment, see [Custom Website Deployment Scripts with Kudu].</span></span>

### <a name="using-a-packagejson-file"></a><span data-ttu-id="20a3c-130">Utilizzare un file package.json</span><span class="sxs-lookup"><span data-stu-id="20a3c-130">Using a package.json file</span></span>
