## <a name="how-alerts-work"></a><span data-ttu-id="14cca-142">Funzionamento degli avvisi</span><span class="sxs-lookup"><span data-stu-id="14cca-142">How alerts work</span></span>

* <span data-ttu-id="14cca-145">Quando lo stato di un avviso viene modificato, viene generata una notifica.</span><span class="sxs-lookup"><span data-stu-id="14cca-145">A notification is generated when an alert changes state.</span></span> <span data-ttu-id="14cca-146">Se la condizione dell'avviso era già impostata su true durante la creazione dell'avviso, è possibile che non si riceva alcuna notifica finché la condizione non viene impostata su false.</span><span class="sxs-lookup"><span data-stu-id="14cca-146">(If the alert condition was already true when you created the alert, you might not get a notification until the condition goes false.)</span></span>
* <span data-ttu-id="14cca-147">Ogni notifica genera un messaggio di posta elettronica se è stata selezionata la casella dei messaggi di posta elettronica o sono stati specificati indirizzi di posta elettronica.</span><span class="sxs-lookup"><span data-stu-id="14cca-147">Each notification generates an email if you checked the emails box, or provided email addresses.</span></span> <span data-ttu-id="14cca-148">È anche possibile esaminare l'elenco di riepilogo a discesa Notifiche.</span><span class="sxs-lookup"><span data-stu-id="14cca-148">You can also look at the Notifications drop-down list.</span></span>
* <span data-ttu-id="14cca-149">Un avviso viene valutato ogni volta che arriva una metrica, ma non altrimenti.</span><span class="sxs-lookup"><span data-stu-id="14cca-149">An alert is evaluated each time a metric arrives, but not otherwise.</span></span>
* <span data-ttu-id="14cca-150">La valutazione aggrega la metrica per il periodo precedente e quindi la confronta con la soglia per determinare il nuovo stato.</span><span class="sxs-lookup"><span data-stu-id="14cca-150">The evaluation aggregates the metric over the preceding period, and then compares it to the threshold to determine the new state.</span></span>
* <span data-ttu-id="14cca-151">Il periodo scelto specifica l'intervallo per il quale vengono aggregate le metriche.</span><span class="sxs-lookup"><span data-stu-id="14cca-151">The period that you choose specifies the interval over which metrics are aggregated.</span></span> <span data-ttu-id="14cca-152">Non influisce sulla frequenza con cui viene valutato l'avviso: ciò dipende dalla frequenza di arrivo delle metriche.</span><span class="sxs-lookup"><span data-stu-id="14cca-152">It doesn't affect how often the alert is evaluated: that depends on the frequency of arrival of metrics.</span></span>
* <span data-ttu-id="14cca-153">Se per un periodo di tempo non arrivano dati per una determinata metrica, il divario comporta diversi effetti sulla valutazione dell'avviso e sui grafici in Esplora metriche.</span><span class="sxs-lookup"><span data-stu-id="14cca-153">If no data arrives for a particular metric for some time, the gap has different effects on alert evaluation and on the charts in metric explorer.</span></span> <span data-ttu-id="14cca-154">Se in Esplora metriche non viene visualizzato alcun dato per un periodo più lungo dell'intervallo di campionamento del grafico, il grafico mostra un valore pari a 0.</span><span class="sxs-lookup"><span data-stu-id="14cca-154">In metric explorer, if no data is seen for longer than the chart's sampling interval, the chart shows a value of 0.</span></span> <span data-ttu-id="14cca-155">Un avviso basato sulla metrica stessa non viene valutato nuovamente e lo stato dell'avviso rimane invariato.</span><span class="sxs-lookup"><span data-stu-id="14cca-155">But an alert based on the same metric is not be reevaluated, and the alert's state remains unchanged.</span></span> 
  
    <span data-ttu-id="14cca-156">Quando infine arrivano i dati, il grafico viene riportato su un valore diverso da zero.</span><span class="sxs-lookup"><span data-stu-id="14cca-156">When data eventually arrives, the chart jumps back to a non-zero value.</span></span> <span data-ttu-id="14cca-157">L'avviso viene valutato in base ai dati disponibili per il periodo specificato.</span><span class="sxs-lookup"><span data-stu-id="14cca-157">The alert evaluates based on the data available for the period you specified.</span></span> <span data-ttu-id="14cca-158">Se il nuovo punto dati è l'unico disponibile nel periodo, la funzione di aggregazione si basa solo su quel punto dati.</span><span class="sxs-lookup"><span data-stu-id="14cca-158">If the new data point is the only one available in the period, the aggregate is based just on that data point.</span></span>
* <span data-ttu-id="14cca-159">Un avviso può spesso passare velocemente dallo stato di avviso a quello integro e viceversa, anche se si imposta un periodo prolungato.</span><span class="sxs-lookup"><span data-stu-id="14cca-159">An alert can flicker frequently between alert and healthy states, even if you set a long period.</span></span> <span data-ttu-id="14cca-160">Questa situazione può verificarsi se il valore della metrica si aggira intorno alla soglia.</span><span class="sxs-lookup"><span data-stu-id="14cca-160">This can happen if the metric value hovers around the threshold.</span></span> <span data-ttu-id="14cca-161">Non esiste alcuna isteresi nella soglia: la transizione allo stato di avviso si verifica in corrispondenza dello stesso valore della transizione allo stato integro.</span><span class="sxs-lookup"><span data-stu-id="14cca-161">There is no hysteresis in the threshold: the transition to alert happens at the same value as the transition to healthy.</span></span>
