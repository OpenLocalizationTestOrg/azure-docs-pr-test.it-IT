## <a name="how-it-works"></a><span data-ttu-id="7cf03-113">Funzionamento</span><span class="sxs-lookup"><span data-stu-id="7cf03-113">How it works</span></span>
![Panoramica della rete CDN](./media/cdn-overview/cdn-overview.png)

1. <span data-ttu-id="7cf03-115">Un utente (Alice) richiede un file, detto anche asset, usando un URL con un nome di dominio particolare, ad esempio `<endpointname>.azureedge.net`.</span><span class="sxs-lookup"><span data-stu-id="7cf03-115">A user (Alice) requests a file (also called an asset) using a URL with a special domain name, such as `<endpointname>.azureedge.net`.</span></span>  <span data-ttu-id="7cf03-116">Il servizio DNS instrada la richiesta alla località POP (Point of Presence) che offre le migliori prestazioni.</span><span class="sxs-lookup"><span data-stu-id="7cf03-116">DNS routes the request to the best performing Point-of-Presence (POP) location.</span></span>  <span data-ttu-id="7cf03-117">In genere questo è il POP geograficamente più vicino all'utente.</span><span class="sxs-lookup"><span data-stu-id="7cf03-117">Usually this is the POP that is geographically closest to the user.</span></span>
2. <span data-ttu-id="7cf03-118">Se nella cache dei server perimetrali del POP il file non è disponibile, verrà richiesto automaticamente all'origine.</span><span class="sxs-lookup"><span data-stu-id="7cf03-118">If the edge servers in the POP do not have the file in their cache, the edge server requests the file from the origin.</span></span>  <span data-ttu-id="7cf03-119">L'origine può essere un'app Web di Azure, il servizio Cloud di Azure, un account di archiviazione di Azure o qualsiasi server Web accessibile pubblicamente.</span><span class="sxs-lookup"><span data-stu-id="7cf03-119">The origin can be an Azure Web App, Azure Cloud Service, Azure Storage account, or any publicly accessible web server.</span></span>
3. <span data-ttu-id="7cf03-120">L'origine restituisce il file al server perimetrale, comprese le intestazioni HTTP facoltative che descrivono la durata (TTL) del file.</span><span class="sxs-lookup"><span data-stu-id="7cf03-120">The origin returns the file to the edge server, including optional HTTP headers describing the file's Time-to-Live (TTL).</span></span>
4. <span data-ttu-id="7cf03-121">Il server perimetrale memorizza il file nella cache e lo restituisce al richiedente originale (Alice).</span><span class="sxs-lookup"><span data-stu-id="7cf03-121">The edge server caches the file and returns the file to the original requestor (Alice).</span></span>  <span data-ttu-id="7cf03-122">Il file rimane nella cache del server perimetrale fino alla scadenza del valore TTL.</span><span class="sxs-lookup"><span data-stu-id="7cf03-122">The file remains cached on the edge server until the TTL expires.</span></span>  <span data-ttu-id="7cf03-123">Se l'origine ha specificato un valore TTL, il valore predefinito è di sette giorni.</span><span class="sxs-lookup"><span data-stu-id="7cf03-123">If the origin didn't specify a TTL, the default TTL is seven days.</span></span>
5. <span data-ttu-id="7cf03-124">Altri utenti possono quindi richiedere lo stesso file usando lo stesso URL e anche essere indirizzati allo stesso POP.</span><span class="sxs-lookup"><span data-stu-id="7cf03-124">Additional users may then request the same file using that same URL, and may also be directed to that same POP.</span></span>
6. <span data-ttu-id="7cf03-125">Se il valore TTL per il file non è ancora scaduto, il server perimetrale restituisce il file dalla cache,</span><span class="sxs-lookup"><span data-stu-id="7cf03-125">If the TTL for the file hasn't expired, the edge server returns the file from the cache.</span></span>  <span data-ttu-id="7cf03-126">offrendo quindi un'esperienza utente più veloce ed efficiente.</span><span class="sxs-lookup"><span data-stu-id="7cf03-126">This results in a faster, more responsive user experience.</span></span>
