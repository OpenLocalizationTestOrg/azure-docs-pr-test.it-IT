## <a name="overview"></a><span data-ttu-id="252d2-104">Panoramica</span><span class="sxs-lookup"><span data-stu-id="252d2-104">Overview</span></span>
<span data-ttu-id="252d2-105">Il database SQL di Azure gestisce le risorse disponibili per un database usando due meccanismi diversi: la **governance delle risorse** e l'**imposizione di limiti**.</span><span class="sxs-lookup"><span data-stu-id="252d2-105">Azure SQL Database manages the resources available to a database using two different mechanisms: **Resources Governance** and **Enforcement of Limits**.</span></span> <span data-ttu-id="252d2-106">Questo argomento illustra queste due aree principali relative alla gestione delle risorse.</span><span class="sxs-lookup"><span data-stu-id="252d2-106">This topic explains these two main areas of resource management.</span></span>

## <a name="resource-governance"></a><span data-ttu-id="252d2-107">Governance delle risorse</span><span class="sxs-lookup"><span data-stu-id="252d2-107">Resource governance</span></span>
<span data-ttu-id="252d2-108">Uno degli obiettivi di progettazione dei piani tariffari Basic, Standard, Premium e Premium RS prevede che il database SQL di Azure si comporti come se il database fosse in esecuzione in un computer dedicato, indipendente dagli altri database.</span><span class="sxs-lookup"><span data-stu-id="252d2-108">One of the design goals of the Basic, Standard, Premium, and Premium RS service tiers is for Azure SQL Database to behave as if the database is running on its own machine, isolated from other databases.</span></span> <span data-ttu-id="252d2-109">La governance delle risorse emula questo comportamento.</span><span class="sxs-lookup"><span data-stu-id="252d2-109">Resource governance emulates this behavior.</span></span> <span data-ttu-id="252d2-110">Se l'utilizzo delle risorse aggregato raggiunge il numero massimo di risorse disponibili di CPU, memoria, I/O di log e I/O di dati assegnate al database, la governance delle risorse inserisce nella coda le query in esecuzione e assegna le risorse alle query accodate man mano che si liberano.</span><span class="sxs-lookup"><span data-stu-id="252d2-110">If the aggregated resource utilization reaches the maximum available CPU, Memory, Log I/O, and Data I/O resources assigned to the database, resource governance queues queries in execution and assign resources to the queued queries as they free up.</span></span>

<span data-ttu-id="252d2-111">Come in un computer dedicato, l'utilizzo di tutte le risorse disponibili determina un allungamento dei tempi per le query attualmente in esecuzione che può comportare timeout dei comandi nel client.</span><span class="sxs-lookup"><span data-stu-id="252d2-111">As on a dedicated machine, utilizing all available resources results in a longer execution of currently executing queries, which can result in command timeouts on the client.</span></span> <span data-ttu-id="252d2-112">Nelle applicazioni con una logica di ripetizione dei tentativi aggressiva e nelle applicazioni che eseguono query sul database con frequenza elevata possono essere restituiti messaggi di errore durante il tentativo di eseguire nuove query quando è stato raggiunto il limite di richieste simultanee.</span><span class="sxs-lookup"><span data-stu-id="252d2-112">Applications with aggressive retry logic and applications that execute queries against the database with a high frequency can encounter errors messages when trying to execute new queries when the limit of concurrent requests has been reached.</span></span>

### <a name="recommendations"></a><span data-ttu-id="252d2-113">Consigli:</span><span class="sxs-lookup"><span data-stu-id="252d2-113">Recommendations:</span></span>
<span data-ttu-id="252d2-114">Monitorare l'utilizzo delle risorse e i tempi di risposta medi delle query quando si sta per raggiungere l'utilizzo massimo di un database.</span><span class="sxs-lookup"><span data-stu-id="252d2-114">Monitor the resource utilization and the average response times of queries when nearing the maximum utilization of a database.</span></span> <span data-ttu-id="252d2-115">Quando si verificano latenze di query più elevate, sono disponibili tre opzioni:</span><span class="sxs-lookup"><span data-stu-id="252d2-115">When encountering higher query latencies you generally have three options:</span></span>

3. <span data-ttu-id="252d2-118">Ottimizzare le query per ridurre l'utilizzo delle risorse di ogni query.</span><span class="sxs-lookup"><span data-stu-id="252d2-118">Optimize queries to reduce the resource utilization of each query.</span></span> <span data-ttu-id="252d2-119">Per altre informazioni, vedere la sezione Hint/ottimizzazione di query in Linee guida per le prestazioni del database SQL di Azure.</span><span class="sxs-lookup"><span data-stu-id="252d2-119">For more information, see the Query Tuning/Hinting section in the Azure SQL Database Performance Guidance article.</span></span>

## <a name="enforcement-of-limits"></a><span data-ttu-id="252d2-120">imposizione di limiti</span><span class="sxs-lookup"><span data-stu-id="252d2-120">Enforcement of limits</span></span>
<span data-ttu-id="252d2-121">Per le risorse diverse da CPU, memoria, I/O di log e I/O di dati, al raggiungimento dei limiti le nuove richieste vengono negate.</span><span class="sxs-lookup"><span data-stu-id="252d2-121">Resources other than CPU, Memory, Log I/O, and Data I/O are enforced by denying new requests when limits are reached.</span></span> <span data-ttu-id="252d2-122">Quando un database raggiunge le dimensioni massime configurate, gli inserimenti e gli aggiornamenti che aumentano le dimensioni dei dati hanno esito negativo, mentre le selezioni e le eliminazioni continuano a funzionare.</span><span class="sxs-lookup"><span data-stu-id="252d2-122">When a database reaches the configured maximum size limit, inserts and updates that increase data size fail, while selects and deletes continue to work.</span></span> <span data-ttu-id="252d2-123">I client visualizzano un [messaggio di errore](sql-database-develop-error-messages.md) a seconda del limite che è stato raggiunto.</span><span class="sxs-lookup"><span data-stu-id="252d2-123">Clients receive an [error message](sql-database-develop-error-messages.md) depending on the limit that has been reached.</span></span>

<span data-ttu-id="252d2-124">Ad esempio, il numero di connessioni a un database SQL e il numero di richieste simultanee che possono essere elaborate sono limitati.</span><span class="sxs-lookup"><span data-stu-id="252d2-124">For example, the number of connections to a SQL database and the number of concurrent requests that can be processed are restricted.</span></span> <span data-ttu-id="252d2-125">Nel database SQL, il numero di connessioni al database può essere maggiore del numero di richieste simultanee per supportare i pool di connessioni.</span><span class="sxs-lookup"><span data-stu-id="252d2-125">SQL Database allows the number of connections to the database to be greater than the number of concurrent requests to support connection pooling.</span></span> <span data-ttu-id="252d2-126">Mentre il numero di connessioni disponibili può essere facilmente controllato dall'applicazione, il numero di richieste parallele è più difficile da stimare e da controllare.</span><span class="sxs-lookup"><span data-stu-id="252d2-126">While the number of connections that are available can easily be controlled by the application, the number of parallel requests is often times harder to estimate and to control.</span></span> <span data-ttu-id="252d2-127">In particolare, durante i carichi di picco quando l'applicazione invia troppe richieste o il database raggiunge i relativi limiti e inizia ad accumulare thread di lavoro a causa dei tempi di esecuzione delle query prolungati, è possibile che si verifichino errori.</span><span class="sxs-lookup"><span data-stu-id="252d2-127">Especially during peak loads when the application either sends too many requests or the database reaches its resource limits and starts piling up worker threads due to longer running queries, errors can be encountered.</span></span>
