## <a name="connector"></a><span data-ttu-id="b5d4a-115">Connettore</span><span class="sxs-lookup"><span data-stu-id="b5d4a-115">Connector</span></span>

<span data-ttu-id="b5d4a-118">I connettori offrono la possibilità di comunicare senza agente usando protocolli di sistema remoto invece di basarsi sulla distribuzione di agenti specializzati.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-118">The connectors provide the agentless ability to converse by using remote system protocols instead of relying on the deployment of specialized agents.</span></span> <span data-ttu-id="b5d4a-119">Ciò comporta una riduzione del rischio e dei tempi di distribuzione, in particolare in caso di applicazioni e sistemi critici.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-119">This means decreased risk and deployment times, especially when dealing with critical applications and systems.</span></span>

<span data-ttu-id="b5d4a-122">Le importazioni e le esportazioni sono eseguite solo quando pianificato, permettendo un maggiore isolamento dalle modifiche apportate nel sistema, poiché le modifiche non vengono propagate automaticamente all'origine dati connessa.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-122">Imports and exports only occur when scheduled, allowing for further insulation from changes occurring within the system, since changes do not automatically propagate to the connected data source.</span></span> <span data-ttu-id="b5d4a-123">Gli sviluppatori, inoltre, possono creare connettori personalizzati per connettersi praticamente a qualsiasi origine dati.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-123">In addition, developers may also create their own connectors for connecting to virtually any data source.</span></span>

## <a name="connector-space"></a><span data-ttu-id="b5d4a-134">Spazio connettore</span><span class="sxs-lookup"><span data-stu-id="b5d4a-134">Connector space</span></span>

<span data-ttu-id="b5d4a-136">Ciò permette al servizio di sincronizzazione di funzionare localmente, senza che sia necessario contattare il sistema remoto durante la sincronizzazione di oggetti, e limita le interazioni alle sole importazioni ed esportazioni.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-136">This allows the sync service to operate locally without the need to contact the remote system when synchronizing the objects and restricts interaction to imports and exports only.</span></span>

<span data-ttu-id="b5d4a-137">Quando l'origine dati e il connettore sono in grado di fornire un elenco di modifiche (importazione delta), l'efficienza operativa aumenta in modo drastico, poiché saranno scambiate solo le modifiche successive all'ultimo ciclo di polling.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-137">When the data source and the connector have the ability to provide a list of changes (a delta import), then the operational efficiency increases dramatically as only changes since the last polling cycle are exchanged.</span></span> <span data-ttu-id="b5d4a-138">Lo spazio connettore isola l'origine dati connessa dalla propagazione automatica delle modifiche, richiedendo che il connettore pianifichi le importazioni e le esportazioni.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-138">The connector space insulates the connected data source from changes propagating automatically by requiring that the connector schedule imports and exports.</span></span> <span data-ttu-id="b5d4a-139">Ciò offre una maggiore sicurezza durante il test, l'anteprima o la conferma dell'aggiornamento successivo.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-139">This added insurance grants you peace of mind while testing, previewing, or confirming the next update.</span></span>

## <a name="metaverse"></a><span data-ttu-id="b5d4a-140">Metaverse</span><span class="sxs-lookup"><span data-stu-id="b5d4a-140">Metaverse</span></span>
<span data-ttu-id="b5d4a-141">Per metaverse si intende la visualizzazione consolidata di tutte le identità unite dagli spazi connettore vicini.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-141">The metaverse is the consolidated view of all joined identities from neighboring connector spaces.</span></span>

<span data-ttu-id="b5d4a-142">Poiché le identità sono collegate tra loro e l'autorità è assegnata a diversi attributi tramite i mapping di flusso di importazione, l'oggetto metaverse centrale inizia ad aggregare informazioni da più sistemi.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-142">As identities are linked together and authority is assigned for various attributes through import flow mappings, the central metaverse object begins to aggregate information from multiple systems.</span></span> <span data-ttu-id="b5d4a-143">Da questo flusso di attributi dell'oggetto, i mapping portano le informazioni ai sistemi in uscita.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-143">From this object attribute flow, mappings carry information to outbound systems.</span></span>

<span data-ttu-id="b5d4a-144">Gli oggetti sono creati quando un sistema autorevole li proietta nel metaverse.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-144">Objects are created when an authoritative system projects them into the metaverse.</span></span> <span data-ttu-id="b5d4a-145">Non appena vengono rimosse tutte le connessioni, l'oggetto metaverse sarà eliminato.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-145">As soon as all connections are removed, the metaverse object is deleted.</span></span>

<span data-ttu-id="b5d4a-146">Gli oggetti nel metaverse non possono essere eliminati direttamente.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-146">Objects in the metaverse cannot be edited directly.</span></span> <span data-ttu-id="b5d4a-147">Tutti i dati nell'oggetto devono essere forniti tramite il flusso di attributi.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-147">All data in the object must be contributed through attribute flow.</span></span> <span data-ttu-id="b5d4a-148">Il metaverse mantiene connettori permanenti con ogni spazio connettore.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-148">The metaverse maintains persistent connectors with each connector space.</span></span> <span data-ttu-id="b5d4a-149">Questi connettori non richiedono una nuova valutazione per ogni esecuzione della sincronizzazione.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-149">These connectors do not require reevaluation for each synchronization run.</span></span> <span data-ttu-id="b5d4a-150">Ciò significa che il servizio di sincronizzazione Azure AD Connect non deve individuare ogni volta l'oggetto remoto corrispondente.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-150">This means that Azure AD Connect sync does not have to locate the matching remote object each time.</span></span> <span data-ttu-id="b5d4a-151">Sarà quindi possibile evitare la necessità di usare agenti costosi per impedire modifiche ad attributi che sarebbero normalmente responsabili della correlazione degli oggetti.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-151">This avoids the need for costly agents to prevent changes to attributes that would normally be responsible for correlating the objects.</span></span>

<span data-ttu-id="b5d4a-152">Quando rileva nuove origini dati che potrebbero includere oggetti preesistenti da gestire, il servizio di sincronizzazione Azure AD Connect usa un processo definito regola di join per valutare candidati potenziali con cui stabilire un collegamento.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-152">When discovering new data sources that may have preexisting objects that need to be managed, Azure AD Connect sync uses a process called a join rule to evaluate potential candidates with which to establish a link.</span></span>
<span data-ttu-id="b5d4a-153">Dopo la creazione del collegamento, la valutazione non viene ripetuta ed è permesso il flusso normale dell'attributo tra l'origine dati remota connessa e il metaverse.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-153">Once the link is established, this evaluation does not reoccur and normal attribute flow can occur between the remote connected data source and the metaverse.</span></span>

## <a name="provisioning"></a><span data-ttu-id="b5d4a-154">Provisioning</span><span class="sxs-lookup"><span data-stu-id="b5d4a-154">Provisioning</span></span>
<span data-ttu-id="b5d4a-155">Quando un'origine autorevole proietta un nuovo oggetto nel metaverse, un nuovo oggetto spazio connettore può essere creato in un altro connettore che rappresenta un'origine dati connessa successiva.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-155">When an authoritative source projects a new object into the metaverse a new connector space object can be created in another Connector representing a downstream connected data source.</span></span>

<span data-ttu-id="b5d4a-156">In questo modo sarà stabilito implicitamente un collegamento e il flusso dell'attributo potrà procedere in modo bidirezionale.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-156">This inherently establishes a link, and attribute flow can proceed bi-directionally.</span></span>

<span data-ttu-id="b5d4a-157">Quando una regola determina che è necessario creare un nuovo oggetto spazio connettore, questa operazione è definita provisioning.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-157">Whenever a rule determines that a new connector space object needs to be created, it is called provisioning.</span></span> <span data-ttu-id="b5d4a-158">Poiché tuttavia questa operazione si verifica solo nello spazio connettore, verrà applicata all'origine dati connessa solo quando sarà eseguita un'esportazione.</span><span class="sxs-lookup"><span data-stu-id="b5d4a-158">However, because this operation only takes place within the connector space, it does not carry over into the connected data source until an export is performed.</span></span>
