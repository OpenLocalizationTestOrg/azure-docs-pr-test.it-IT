  * <span data-ttu-id="19834-132">Nell'**apprendimento supervisionato** ogni punto dati è etichettato con o associato a una categoria o un valore di interesse.</span><span class="sxs-lookup"><span class="sxs-lookup"><span data-stu-id="19834-137">This is a popular and useful type of machine learning.</span></span> <span data-ttu-id="19834-138">Tutti i moduli di Azure Machine Learning sono algoritmi di apprendimento supervisionato, ad eccezione di [K-Means Clustering][k-means-clustering].</span><span class="sxs-lookup"><span data-stu-id="19834-138">All of the modules in Azure Machine   * <span data-ttu-id="19834-142">Nell'**apprendimento per rinforzo** l'algoritmo arriva a scegliere un'azione in risposta a ogni punto dati.</span><span class="sxs-lookup"><span data-stu-id="19834-142">In **reinforcement learning**, the algorithm gets to choose an action in response to each data point.</span></span> <span data-ttu-id="19834-143">Si tratta di un approccio comune in robotica, in cui il set di letture del sensore in un certo momento è un punto dati e l'algoritmo deve scegliere l'azione successiva del robot.</span><span class="sxs-lookup"><span data-stu-id="19834-143">It is a common approach in robotics, where the set of sensor readings at one point in time is a data point, and the algorithm must choose the robot’s next action.</span></span> <span data-ttu-id="19834-144">Questo approccio è ideale anche per applicazioni "Internet delle cose" (Internet of Things, IoT).</span><span class="sxs-lookup"><span data-stu-id="19834-144">It's also a natural fit for Internet of Things applications.</span></span> <span data-ttu-id="19834-145">L'algoritmo di apprendimento riceve anche un segnale di ricompensa poco dopo, a indicare il livello di correttezza della decisione presa.</span><span class="sxs-lookup"><span data-stu-id="19834-145">The learning algorithm also receives a reward signal a short time later, indicating how good the decision was.</span></span> <span data-ttu-id="19834-146">In base a questo segnale, l'algoritmo modifica la propria strategia per ottenere la ricompensa maggiore.</span><span class="sxs-lookup"><span data-stu-id="19834-146">Based on this, the algorithm modifies its strategy in order to achieve the highest reward.</span></span> <span data-ttu-id="19834-147">Attualmente in Azure ML non sono disponibili moduli di algoritmi di apprendimento per rinforzo.</span><span class="sxs-lookup"><span data-stu-id="19834-147">Currently there are no reinforcement learning algorithm modules in Azure ML.</span></span>

* <span data-ttu-id="19834-148">I **metodi bayesiani** presuppongono punti dati statisticamente indipendenti.</span><span class="sxs-lookup"><span data-stu-id="19834-148">**Bayesian methods** make the assumption of statistically independent data points.</span></span> <span data-ttu-id="19834-149">Questo significa che la variabilità non modellata in un punto dati non è correlata agli altri, ovvero non può essere stimata.</span><span class="sxs-lookup"><span data-stu-id="19834-149">This means that the unmodeled variability in one data point is uncorrelated with others, that is, it can’t be predicted.</span></span> <span data-ttu-id="19834-150">Ad esempio, se i dati registrati sono il numero di minuti fino all'arrivo del prossimo treno della metropolitana, due misurazioni eseguite a distanza di un giorno sono statisticamente indipendenti.</span><span class="sxs-lookup"><span data-stu-id="19834-150">For example, if the data being recorded is the number of minutes until the next subway train arrives, two measurements taken a day apart are statistically independent.</span></span> <span data-ttu-id="19834-151">Tuttavia, due misurazioni eseguite a distanza di un minuto non sono statisticamente indipendenti e il valore di una è altamente predittivo rispetto a quello dell'altra.</span><span class="sxs-lookup"><span data-stu-id="19834-151">However, two measurements taken a minute apart are not statistically independent - the value of one is highly predictive of the value of the other.</span></span>

* <span data-ttu-id="19834-152">La **regressione con alberi delle decisioni con boosting** sfrutta la sovrapposizione delle caratteristiche o l'interazione tra caratteristiche.</span><span class="sxs-lookup"><span data-stu-id="19834-152">**Boosted decision tree regression** takes advantage of feature overlap or interaction among features.</span></span> <span data-ttu-id="19834-153">Questo significa che, in ogni punto dati specifico, il valore di una caratteristica è in parte predittivo del valore di un'altra.</span><span class="sxs-lookup"><span data-stu-id="19834-153">That means that, in any given data point, the value of one feature is somewhat predictive of the value of another.</span></span> <span data-ttu-id="19834-154">Ad esempio, nel caso di dati di bassa/alta temperatura giornaliera, la possibilità di determinare la bassa temperatura del giorno permette di stimare ragionevolmente anche quella alta.</span><span class="sxs-lookup"><span data-stu-id="19834-154">For example, in daily high/low temperature data, knowing the low temperature for the day allows you to make a reasonable guess at the high.</span></span> <span data-ttu-id="19834-155">Le informazioni contenute nelle due caratteristiche sono in parte ridondanti.</span><span class="sxs-lookup"><span data-stu-id="19834-155">The information contained in the two features is somewhat redundant.</span></span>

* <span data-ttu-id="19834-156">La classificazione di dati in più di due categorie può essere eseguita usando un classificatore essenzialmente multiclasse o combinando un set di classificatori a due classi in un **insieme**.</span><span class="sxs-lookup"><span data-stu-id="19834-156">Classifying data into more than two categories can be done by either using an inherently multi-class classifier, or by combining a set of two-class classifiers into an **ensemble**.</span></span> <span data-ttu-id="19834-157">L'approccio basato su un insieme prevede un classificatore a due classi per ogni classe, ognuno dei quali separa i dati in due categorie: "questa classe" e "non questa classe".</span><span class="sxs-lookup"><span data-stu-id="19834-157">In the ensemble approach, there is a separate two-class classifier for each class - each one separates the data into two categories:  “this class” and “not this class.”</span></span> <span data-ttu-id="19834-158">I classificatori votano quindi l'assegnazione corrente del punto dati.</span><span class="sxs-lookup"><span data-stu-id="19834-158">Then these classifiers vote on the correct assignment of the data point.</span></span> <span data-ttu-id="19834-159">Questo è il principio operativo alla base del modulo [One-vs-All Multiclass][one-vs-all-multiclass].</span><span class="sxs-lookup"><span data-stu-id="19834-159">This is the operational principle behind [One-vs-All Multiclass][one-vs-all-multiclass].</span></span>

* <span data-ttu-id="19834-160">Diversi metodi, tra cui la regressione logistica e Bayes Point Machine, presuppongono **limiti tra classi lineari**,</span><span class="sxs-lookup"><span data-stu-id="19834-160">Several methods, including logistic regression and the Bayes point machine, assume **linear class boundaries**.</span></span> <span data-ttu-id="19834-161">ovvero assumono che i limiti tra le classi siano approssimativamente linee rette (oppure iperpiani nel caso più generale).</span><span class="sxs-lookup"><span data-stu-id="19834-161">That is, they assume that the boundaries between classes are approximately straight lines (or hyperplanes in the more general case).</span></span> <span data-ttu-id="19834-162">Spesso questa caratteristica dei dati non viene individuata fino a dopo aver provato a separarli, ma può essere determinata in genere visualizzando i dati in anticipo.</span><span class="sxs-lookup"><span data-stu-id="19834-162">Often this is a characteristic of the data that you don’t know until after you’ve tried to separate it, but it’s something that typically can be learned by visualizing beforehand.</span></span> <span data-ttu-id="19834-163">Se i limiti di classe appaiono molto irregolari, optare per alberi delle decisioni, giungle delle decisioni, macchine a vettori di supporto o reti neurali.</span><span class="sxs-lookup"><span data-stu-id="19834-163">If the class boundaries look very irregular, stick with decision trees, decision jungles, support vector machines, or neural networks.</span></span>
