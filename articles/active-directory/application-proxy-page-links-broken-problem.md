## <a name="overview"></a><span data-ttu-id="6c08e-105">Panoramica</span><span class="sxs-lookup"><span data-stu-id="6c08e-105">Overview</span></span> 
<span data-ttu-id="6c08e-106">Dopo aver pubblicato un'app proxy di applicazione, gli unici collegamenti che per impostazione predefinita funzionano nell'applicazione sono i collegamenti a destinazioni contenute all'interno dell'URL radice pubblicato.</span><span class="sxs-lookup"><span data-stu-id="6c08e-106">After publishing an Application Proxy app, the only links that work by default in the application are links to destinations contained within the published root URL.</span></span> <span data-ttu-id="6c08e-107">I collegamenti all'interno delle applicazioni non funzionano probabilmente perché l'URL interno per l'applicazione non include tutte le destinazioni dei collegamenti nell'applicazione.</span><span class="sxs-lookup"><span data-stu-id="6c08e-107">The links within the applications aren’t working, the internal URL for the application probably does not include all the destinations of links within the application.</span></span>

<span data-ttu-id="6c08e-108">**Perché si verifica questo problema?**</span><span class="sxs-lookup"><span data-stu-id="6c08e-108">**Why does this happen?**</span></span> <span data-ttu-id="6c08e-109">Quando si fa clic su un collegamento in un'applicazione, il proxy di applicazione prova a risolvere l'URL come URL interno nella stessa applicazione oppure come URL disponibile esternamente.</span><span class="sxs-lookup"><span data-stu-id="6c08e-109">When clicking a link in an application, Application Proxy tries to resolve the URL as either an internal URL within the same application, or as an externally available URL.</span></span> <span data-ttu-id="6c08e-110">Se il collegamento punta a un URL interno che non è incluso nella stessa applicazione, non appartiene ad alcuno di questi bucket e restituisce un errore di URL non trovato.</span><span class="sxs-lookup"><span data-stu-id="6c08e-110">If the link points to an internal URL that is not within the same application, it does not belong to either of these buckets and result in a not found error.</span></span>

## <a name="ways-you-can-resolve-broken-links"></a><span data-ttu-id="6c08e-111">Come correggere i collegamenti interrotti</span><span class="sxs-lookup"><span data-stu-id="6c08e-111">Ways you can resolve broken links</span></span>

<span data-ttu-id="6c08e-112">È possibile risolvere questo problema in tre modi.</span><span class="sxs-lookup"><span data-stu-id="6c08e-112">There are three ways to resolve this issue.</span></span> <span data-ttu-id="6c08e-113">Le scelte seguenti sono elencate in ordine di complessità crescente.</span><span class="sxs-lookup"><span data-stu-id="6c08e-113">The choices below are in listed in increasing complexity.</span></span>

1.  <span data-ttu-id="6c08e-114">Assicurarsi che l'URL interno sia una radice che contiene tutti i collegamenti rilevanti per l'applicazione.</span><span class="sxs-lookup"><span data-stu-id="6c08e-114">Make sure the internal URL is a root that contains all the relevant links for the application.</span></span> <span data-ttu-id="6c08e-115">In questo modo, tutti i collegamenti possono essere risolti come contenuto pubblicato all'interno della stessa applicazione.</span><span class="sxs-lookup"><span data-stu-id="6c08e-115">This allows all links to be resolved as content published within the same application.</span></span>

    <span data-ttu-id="6c08e-116">Se si modifica l'URL interno, ma non si vuole modificare la pagina di destinazione per gli utenti, modificare l'URL di pagina iniziale nell'URL interno precedentemente pubblicato.</span><span class="sxs-lookup"><span data-stu-id="6c08e-116">If you change the internal URL but don’t want to change the landing page for users, change the Home page URL to the previously published internal URL.</span></span> <span data-ttu-id="6c08e-117">A questo scopo, è possibile passare a "Azure Active Directory" -&gt; Registrazioni per l'app -&gt; selezionare l'applicazione -&gt; Proprietà.</span><span class="sxs-lookup"><span data-stu-id="6c08e-117">This can be done by going to “Azure Active Directory” -&gt; App Registrations -&gt; select the application -&gt; Properties.</span></span> <span data-ttu-id="6c08e-118">In questa scheda delle proprietà è visualizzato il campo "URL pagina iniziale", che può essere impostato come pagina di destinazione desiderata.</span><span class="sxs-lookup"><span data-stu-id="6c08e-118">In this properties tab, you see the field “Home Page URL” which you can adjust to be the desired landing page.</span></span>

2.  <span data-ttu-id="6c08e-119">Se le applicazioni usano nomi di dominio completi (FQDN), usare [domini personalizzati](https://docs.microsoft.com/azure/active-directory/active-directory-application-proxy-custom-domains) per pubblicare le applicazioni.</span><span class="sxs-lookup"><span data-stu-id="6c08e-119">If your applications use fully qualified domain names (FQDNs), use [custom domains](https://docs.microsoft.com/azure/active-directory/active-directory-application-proxy-custom-domains) to publish your applications.</span></span> <span data-ttu-id="6c08e-120">Questa funzionalità permette l'uso dello stesso URL internamente ed esternamente.</span><span class="sxs-lookup"><span data-stu-id="6c08e-120">This feature allows the same URL to be used both internally and externally.</span></span>

    <span data-ttu-id="6c08e-121">Questa opzione garantisce che i collegamenti nell'applicazione siano accessibili esternamente tramite il proxy di applicazione, perché i collegamenti a URL interni nell'applicazione vengono riconosciuti anche esternamente.</span><span class="sxs-lookup"><span data-stu-id="6c08e-121">This option ensures that the links in your application are externally accessible through Application Proxy since the links within the application to internal URLs are also recognized externally.</span></span> <span data-ttu-id="6c08e-122">Tutti i collegamenti devono appartenere a un'applicazione pubblicata.</span><span class="sxs-lookup"><span data-stu-id="6c08e-122">Note that all links still need to belong to a published application.</span></span> <span data-ttu-id="6c08e-123">Con questa opzione, tuttavia, i collegamenti non devono appartenere alla stessa applicazione, ma possono appartenere a più applicazioni.</span><span class="sxs-lookup"><span data-stu-id="6c08e-123">However, with this option the links do not need to belong to the same application and can belong to multiple applications.</span></span>

3.  <span data-ttu-id="6c08e-124">Se nessuna di queste opzioni è applicabile, è possibile iscriversi all'anteprima di una nuova funzionalità che esegue la conversione e la riscrittura degli URL.</span><span class="sxs-lookup"><span data-stu-id="6c08e-124">If neither of these options are feasible, you join the preview for a new feature that do URL translation/rewriting.</span></span> <span data-ttu-id="6c08e-125">Con questa opzione, gli URL o i collegamenti interni presenti nel corpo HTML delle applicazioni possono essere convertiti, o "mappati", negli URL del proxy di applicazione esterni pubblicati.</span><span class="sxs-lookup"><span data-stu-id="6c08e-125">With this option, internal URLs or links that exist in the HTML body of your applications be translated, or “mapped”, to the published external App Proxy URLs.</span></span> <span data-ttu-id="6c08e-126">Questa opzione funziona solo per collegamenti nel codice HTML o CSS e non è utile se il collegamento viene generato tramite JS.</span><span class="sxs-lookup"><span data-stu-id="6c08e-126">This only works for links in the HTML or CSS, and this not help if your link is generated through JS.</span></span> 

<span data-ttu-id="6c08e-127">Di conseguenza, è consigliabile usare la soluzione con [domini personalizzati](https://docs.microsoft.com/azure/active-directory/active-directory-application-proxy-custom-domains), se possibile.</span><span class="sxs-lookup"><span data-stu-id="6c08e-127">As a result, we strongly recommend using the [custom domains](https://docs.microsoft.com/azure/active-directory/active-directory-application-proxy-custom-domains) solution if possible.</span></span> <span data-ttu-id="6c08e-128">Per iscriversi all'anteprima della funzionalità, inviare un messaggio di posta elettronica all'indirizzo <aadapfeedback@microsoft.com>, indicando gli ID applicazione.</span><span class="sxs-lookup"><span data-stu-id="6c08e-128">If you do want to join the preview, email <aadapfeedback@microsoft.com> with the applicationId(s).</span></span>
