## <a name="cluster-support"></a>Supporto di cluster
Azure Backup Server proteggere i dati in hello seguenti applicazioni in cluster:

-   File server

-   SQL Server

-   Hyper-V - se si protegge un cluster Hyper-V Usa la protezione di DPM di scalabilità orizzontale, è possibile aggiungere una protezione secondaria per i carichi di lavoro Hyper-V hello protetto.

    Se si esegue Hyper-V in Windows Server 2008 R2, assicurarsi che l'aggiornamento hello tooinstall descritto nell'articolo [975354](https://support.microsoft.com/en-us/kb/975354).
    Se si esegue Hyper-V in Windows Server 2008 R2 in una configurazione cluster, assicurarsi di installare SP2, come descritto in KB [971394](https://support.microsoft.com/en-us/kb/971394).

-   Exchange Server - Il server di Backup di Azure può proteggere i cluster di dischi non condivisi per le versioni di Exchange Server supportate (replica continua cluster) e può inoltre proteggere Exchange Server configurato per la replica continua locale.

-   SQL Server - Il server di Backup di Azure non supporta il backup dei database di SQL Server ospitati in volumi condivisi cluster (CSV).

Server di Backup di Azure è possibile proteggere i carichi di lavoro che si trovano in hello nello stesso dominio come server DPM hello e in un dominio trusted o figlio. Se si desidera tooprotect origini di dati in gruppi di lavoro o domini non trusted, utilizzare l'autenticazione NTLM o certificati per un singolo server o l'autenticazione del certificato solo per un cluster.
